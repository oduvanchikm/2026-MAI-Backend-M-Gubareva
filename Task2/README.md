## Настроить nginx для отдачи статический файлов из public/

Нужно написать собственный конфиг nginx.conf, в котором будет location public. Во время проверки домашней работы нужно будет скачать при показе ресурс (картинка, видео и т.д.), положить его в папку, на который смотрит данный location и открыть его в браузере.

## Создать WSGI-приложение и запустить его с помощью gunicorn

Приложение должно генерировать с помощью модуля random, string пароли, которые удовлетворяют следующим условиям:

- Длина должна быть не менее 8 и не более 16 символов;
- Среди символов должны присутствовать как минимум: 1 цифра, 1 буква в нижнем регистре, 1 буква в верхнем регистре и 1 символ из следующих: #.,!@&^%* ;
- В конце необходимо сделать time.sleep(0.05) -- заснуть на 50ms; 

## Настроить проксирование запросов на nginx

Должен быть location gunicorn, оформленный через upstream, который будет ходить в WSGI-приложение.

## Измерить производительность nginx и gunicorn c помощью ab или wrk

Нужно сделать три измерения:

- На location public;
- На location gunicorn;
- Напрямую на WSGI-приожение; Нужно подобрать такой rps, увеличив который на 10% появятся таймауты/ошибки.
